# Governance

Prism uses maintainer-led, evidence-based governance.

## Roles

- Contributors submit issues, fixtures, documentation, and code.
- Reviewers consistently review an area and may approve changes there.
- Maintainers merge changes, publish releases, handle security reports, and set
  compatibility and quality policy.

Maintainer status is earned through sustained, trustworthy contribution and is
granted by consensus of current maintainers. Inactive maintainers may move to
emeritus status after six months without project activity.

## Decisions

Routine changes use pull-request review. Changes to public operation semantics,
quality tiers, storage formats, security boundaries, licensing, or governance
require a written design issue and maintainer consensus. When consensus is not
possible, the project lead makes the decision and records the rationale.

Releases require passing tests, documented compatibility impact, checksums,
SBOM, and provenance/signature artifacts. Benchmark claims must link to
reproducible data and name known limitations.
